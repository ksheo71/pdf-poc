# 요구사항

## PDF 표시
- `구성도.pdf`를 읽어 웹 페이지에 **1페이지만** 표시한다.
- 백엔드는 PDF를 PNG으로 렌더링해 서빙하고, 프론트엔드는 이미지 위에 오버레이를 그린다.
- 렌더 스케일: 초기 표시용은 2x, 다운로드(커밋)용은 16x로 생성한다 (다운스케일 시 sharpness 확보).

## 텍스트 영역 추출 및 표시
- PyMuPDF 사용.
- 단어 단위로 추출하되, `get_text("rawdict")`의 문자 좌표를 기준으로 **수평 갭이 폰트 크기의 30% 이상** 벌어지면 분리한다 (표 셀 경계에서 인접 단어가 합쳐지지 않도록).
- 각 단어 영역을 **파란색 1px 점선**의 **텍스트 입력 가능한** 직사각형으로 오버레이한다.
- 직사각형은 텍스트 bbox에서 **상하좌우 0px 마진**을 준다.

## 배경 덮기 및 재작성
- 각 단어의 bbox 주변(상/하/좌/우 4px 스트립)에서 픽셀을 샘플링해 중앙값으로 **배경과 동일한 색상**의 사각형으로 원본 텍스트를 덮는다.
- 원본 텍스트는 `add_redact_annot` + `apply_redactions`로 PDF text layer에서 **실제 제거**한다.
- 덮힌 사각형 위에 **원본과 동일한 속성**(font, size, color)으로 텍스트를 다시 표시한다 (프론트엔드 contenteditable div).

## 폰트 처리
- 원본 PDF에 임베디드된 폰트를 `doc.extract_font()`로 추출해 그대로 사용한다.
- 폰트 속성(family, size, weight, color)은 원본 span과 동일하게 유지한다.
- **숫자 토큰**(숫자·쉼표·마침표·괄호 등만 포함하는 단어)은 문자 단어의 최빈 폰트(예: DotumChe)로 `font_name`을 덮어써서 문자와 동일한 폰트로 표시한다.

## 상호작용 상태
| 상태 | 외곽선 | 조인트 | 편집 가능 |
|---|---|---|---|
| idle | 파란 1px 점선 | 숨김 | 아니오 |
| selected | 빨간 1px 실선 | 상/하/좌/우 빨간 점(6px) | 아니오 |
| editing | 빨간 1px 실선 | 상/하/좌/우 빨간 점 | 예 |

### 전이
- `idle` → 클릭 → `selected`
- `selected` → 클릭 → `editing`
- `idle` → 더블클릭 → `editing`
- 어느 상태이든 **외부 클릭 또는 ESC** → `idle` (selected/editing 구분 없이 곧바로 점선으로 복귀)

### 리사이즈 (joint 드래그)
- 각 joint는 해당 변의 **정중앙**에 위치한다.
- joint를 마우스로 드래그하면 직사각형의 해당 방향 크기가 변경된다.
- `editing` 상태에서도 리사이즈가 가능하다.

### 이동 (방향키)
- `selected` 상태에서 방향키 → 5px 이동.
- `Ctrl + 방향키` → 1px 미세 이동.
- `editing` 상태에서는 방향키가 텍스트 캐럿 이동에 사용된다 (기본 브라우저 동작).

## 다운로드 버튼
- 우측 상단에 "PDF 다운로드" 버튼을 배치한다.
- 클릭 시:
  1. 프론트엔드가 **편집된 박스**(텍스트 내용 또는 위치·크기가 초기와 다른 박스)만 선별해 서버에 전송한다.
  2. 서버는 원본 PDF를 새로 열어 편집된 박스의 **원래 위치**를 배경색으로 덮고(redact), 현재 위치에 새 내용을 임베디드 폰트로 다시 삽입한다 (`fitz.TextWriter` + `fitz.Font(fontbuffer=...)`).
  3. 편집되지 않은 박스는 원본 PDF 렌더링이 그대로 보존된다 (pixel-perfect).
  4. 결과 PNG(16x 해상도)를 반환하고, 프론트엔드가 이미지를 교체하면서 **모든 오버레이 박스를 제거**한다.
- 위치 이동 시 baseline도 `origin_y + (new_y0 - orig_y0)`만큼 따라 이동시켜 수직 정렬을 유지한다.

## 기술 스펙
- Frontend: vanilla JS, HTML, Tailwind CSS, shadcn/ui 스타일
- Backend: FastAPI + PyMuPDF (pymupdf)
